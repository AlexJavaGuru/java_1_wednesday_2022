В данный момент, когда вы хотите передать новое поведение
в метод findApples(), вы вынуждены объявить
несколько классов, которые реализуют интерфейс ApplePredicate,
а затем создать несколько экземпляров ApplePredicate объектов,
которые вы используете только один раз при передаче их в качестве
параметра в метод findApples().

Прижётся писать много кода и это трудоемкий процесс!

А есть ли более классное решение?

У Java есть механизм, называемый анонимный класс,
которые позволяют вам объявлять и создавать экземпляр класса одновременно.
Это позволит вам улучшить свой код еще на один шаг,
сделав его немного более лаконичным.
