Дан класс WordsService и метод по подсчёту количества
повторений указанного слова в массиве. Как нам кажется
в реализации этого метода названия для переменной sum
и переметра wordToCount подобраны немного не правильно.
Название переменной sum немного вводит в заблуждение,
сумма чего? Мы же говорим о числе повторений. Попробуйте
придумать другое название для переменной sum, что бы это
название отображало не техническую сторону, а именно бизнес
сущьность этой задачи - подсчёт повторений.
То же самое с параметром wordToCount. Название wordToCount
никак не коррелирует с бизнес смыслом этого параметра - словом
повторения которого нужно подсчитать. Попробуйте придумать
другое название параметру wordToCount.

class WordsService {

    public int countRepetitions(String[] words, String wordToCount) {
        int sum = 0;
        for (String word : words) {
            if (word.equals(wordToCount)) {
                sum++;
            }
        }
        return sum;
    }

}

Возвращаясь к основам. Названия переменных очень важны в
программировании. Это становится очевидным, когда вы начинаете
работать с большими программами в которых много функциональности,
много строчек кода и большое количество классов. Любое классное
название в коде - это как мояк, который укажет вам, что за бизнес
требование реализовано в этом кусочке кода. Если переменная будет
названа плохо, то свет этого мояка будет тусклым и вам придётся
долго разбираться в коде, что бы понять его смысл. И наоборот,
если у переменных в коде будут яркие залинкованные на бизнес
требования названия, то разобраться в этом коде будет намного проще.

Конечно не всегда с первого раза удаётся придумать классные названия
переменным. При объявлении переменной автор даёт ей самое лучшее
название, что пришло ему в голову на тот момент. Продолжая работу
над реализацией поставленной перед вами задачи вы углубляете свое
понимание бизнес области для которой вы решаете задачу. В ходе работы
вам может прийти "озарение" и вы придумаете более выразительное
и яркое название для своей переменной. Вернитесь к своему коду
и для изменения названия переменной воспользуйтесь рефакторингом:
Rename Variable.
