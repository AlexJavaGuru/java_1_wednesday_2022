Генерация следующего поколения в игре Conway's Game of Life
(https://en.wikipedia.org/wiki/Conway%27s_Game_of_Life).

Нужен ли нам отдельный класс для генерации начального состояния игры?
За что будет отвечать этот класс (какие обязанности будет выполнять)?
Почему стоит этот код вынести в отдельный класс и тем самым отделить от остальной системы?

Вспоминаем Single Responsibility Principle!

Генерация начального состояния игры это очень важная функциональность,
так как с неё всё начинается и без неё игра вообще не возможна.

Алгоритм генерации начального состояния может быть совершенно разным:
- рандом
- пользователь задаёт вручную

Поэтому имеет смысл выделить алгоритм генерации начального состояния в отдельную
часть системы и реализовать отдельно от всего остального кода. Для этих целей
идеально подойдёт отдельный класс например с названием GameOfLifeRandomFieldGenerator.

Если мы создадим для кода генерации начального состояния игры отдельный класс,
то его будет просто и легко найти в системе по названию. А так же у нас будет
только одна причина для внесения изменений в код этого класса.
Код этого класса будет меняться
только тогда, когда нужно будет изменить алгоритм генерации начального состояния!
Это идеальная реализация Single Responsibility Principle.
