Создайте класс ArrayService и реализуйте в нём метод
для проверки содержит ли массив целых чисел заданное число.

Перед тем как читать дальше, возьмите чистый лист бумаги и ручку
или карандаш. Ваша задача - придумать сигнатуру метода
для проверки содержит ли массив целых чисел заданное число.

Попробуйте для начала самостоятельно придумать сигнатуру этого метода
и записать ее на бумаге. Чуть позже вы сможете сравнить ее с нашим
вариантом.

Далее приведён список вопросов которые помогут вам поэтапно сформировать
сигнатуру данного метода. Записывайте ответы на бумаге и
пошагово создавайте сигнатуру метода. Начинаем:

Какое название вы дадите этому методу?
Попробуйте придумать несколько подходящих названий, запишите
их на листе бумаги. Выберите среди придуманных вариантов лучший
на ваш взгляд, тот который наилучшим образом передаст смысл того,
что будет делать разрабатываемый вами метод. Попробуйте сделать
так, что бы по одному вашему названию было понятно для чего
предназначен ваш метод, что будет делать ваш метод, какую задачу
будет решать. Помните, что название очень важно. Название это
первое, что увидит другой программист, когда захочет воспользоваться
вашим методом.

Будут ли у этого метода входные параметры?
Если да то сколько и какого типа?
Что бы ответить на эти вопросы подумайте нужно ли будет передать
какие нибудь данные в ваш метод для того, что бы он смог
выполнить свою задачу.

Какие названия вы дадите входным параметрам?
Помните, что названия параметров тоже важная вещь. При вызове
вашего метода среда разработки будет показывать как называются
параметры вашего метода. Их названия тоже должны быть чёткими
и понятными.

Какой результат будет у этого метода? Какие конкретно значения
в качестве результата будет возвращать ваш метод?
Подумайте о результате работы вашего метода. Для этого придумайте
простой тестовый пример с конкретными входными параметрами. Какой
результат должен будет выдать ваш метод при указанных входных параметрах?
Обязательно придумайте позитивный и негативный сценарии, что бы
понять как должен работать и что возвращать в качестве результата
ваш метод в том и другом случае.

Какого типа данных будет возвращаемое значение?
Подумайте, какой тип данных больше всего подходит для того, чтобы
описать диапазон результатов (тоесть все значения, которые
могут быть возвращены в качестве результата работы вашего метода).
Выберите тип данных для описания результата вашего метода.

Теперь пора сложить сигнатуру вашего метода вместе:

    типРезультата названиеМетода(списокПараметровИИхТипов)


Пора сравнить ваш вариант с нашей версией:

    boolean contains(int[] arr, int numberToSearch)

contains - этот глагол часто применяется для проверки есть ли в массиве
           или коллекции данных указанное значение, поэтому мы выбрали
           именно его;
int[] arr - в метод надо передать сам массив в котором будет
            осуществляться поиск;
int numberToSearch - так же входным параметром будет само число,
                     которое надо найти в массиве;
boolean - тип возвращаемого значения:
          true - когда число найдено в массиве,
          false - когда число не найдено в массиве.


Создайте указанный метод в классе ArrayService и напишите
его реализацию.

Создайте класс ArrayServiceTest и напишите автоматические тесты
для разработанного вами метода. Не забудьте написать тесты для
позитивного и негативного сценариев работы вашего метода.
Если при работе над сигнатурой метода вы придумали примеры
входных данных и ожидаемого результата, то вы можете взять их
в качестве примеров и на их основе написать автоматические тесты.
